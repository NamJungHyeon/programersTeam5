# 대피소 찾기 서비스 (SafetyFirst) - Product Requirements Document

## 1. 프로젝트 개요
지역별 대피소 정보를 제공하는 웹 서비스로, 사용자가 주소를 입력하면 가장 가까운 대피소 10곳을 지도 위에 마커로 표시하고 상세 정보를 제공합니다.

## 2. 주요 기능

### 2.1 프론트엔드 (TypeScript + React)
- **카카오 맵 API 연동**: 지도 화면 표시 기능
- **주소 검색**: 검색창을 통한 주소 입력 및 자동완성
- **마커 표시**: 대피소 위치를 지도 위에 마커로 표시
- **대피소 정보 출력**: 선택된 대피소의 상세 정보 표시
- **반응형 UI**: 모바일/데스크톱 대응
- **모듈화된 컴포넌트**: 팀원이 쉽게 수정 가능한 구조

### 2.2 백엔드 (Spring Boot + MySQL)
- **공공데이터 API 연동**: 대피소 정보 수집 및 저장
- **거리 계산 알고리즘**: 사용자 위치 기준 가장 가까운 10곳 선별
- **주소 변환 서비스**: 주소를 좌표로 변환
- **REST API**: 프론트엔드와 데이터 통신
- **데이터베이스 설계**: 대피소 정보 효율적 저장

## 3. 기술 스택
- **Frontend**: TypeScript, React, Kakao Map API
- **Backend**: Spring Boot, MySQL, JPA
- **Tools**: IntelliJ IDEA, Git

## 4. 개발 우선순위
1. 카카오 맵 API 연동 및 기본 지도 화면 구현
2. 마커 표시 기능 구현
3. 검색 UI 컴포넌트 개발
4. 백엔드 API 설계 및 구현
5. 공공데이터 API 연동
6. 거리 계산 및 정렬 로직
7. 대피소 상세 정보 표시
8. 반응형 디자인 적용

## 5. 품질 요구사항
- **유지보수성**: 모듈화된 코드 구조
- **확장성**: 새로운 기능 추가 용이
- **사용성**: 직관적인 UI/UX
- **성능**: 빠른 검색 및 지도 로딩

## 6. 제약사항
- 초보 개발자도 이해하기 쉬운 코드 작성
- 주석을 통한 상세한 설명 제공
- 일관된 코딩 스타일 유지
